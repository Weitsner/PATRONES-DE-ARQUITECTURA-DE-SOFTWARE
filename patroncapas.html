Arquitectura De Software

PATRÓN DE CAPAS

Es un patrón arquitectónico que se puede utilizar para estructurar programas donde las diferentes partes o componentes del sistema se organizan en capas distintas, cada capa tiene una responsabilidad específica y se comunica únicamente con las capas adyacentes, siguiendo una estructura jerárquica.

Las 4 capas más comúnmente encontradas de un sistema de información general son las siguientes:

Capa de presentación (UI): Esta capa es la interfaz de usuario del sistema. Se encarga de interactuar con los usuarios, mostrar datos y recoger información del usuario.

Capa de lógica de negocio (o de dominio): Esta capa contiene la lógica central del negocio o la aplicación. Aquí se implementan las reglas del negocio, algoritmos, validaciones de datos y cualquier otro proceso relacionado con la lógica de la aplicación.

Capa de acceso a datos (DAL): Esta capa es responsable de interactuar con la base de datos u otros sistemas de almacenamiento de datos.

Capa de aplicación (o de servicios): Esta capa proporciona servicios reutilizables que pueden ser utilizados por diferentes partes del sistema.

UTILIDAD

1Aplicaciones web.

2Aplicaciones móviles.

3Sistemas embebidos.

4Sistemas empresariales.

EJEMPLO

Un ejemplo de cómo se puede utilizar el patrón de capas es en el diseño de una aplicación web de comercio electrónico la cual se puede estructurar utilizando tres capas principales: capa de presentación, capa de lógica de negocio y capa de acceso a datos.

Ventajas & Desventajas

VENTAJAS

  • Modularidad.
  • Facilita el desarrollo colaborativo.
  • Reutilización de código.
  • Escalabilidad.

DESVENTAJAS

  • Overhead de comunicación.
  • Acoplamiento.
  • Posible sobrediseño.
